### 1. Create `docs/api-reference.md` (NEW)
Comprehensive API documentation including:

**Authentication**
- `POST /api/auth/login` - JWT authentication

**Driver Check-in/out**
- `POST /api/checkin` - Create check-in with optional idempotency key
- `POST /api/checkin/checkout` - Check out with optional idempotency key

**Admin Endpoints** (require ADMIN or SUPERUSER)
- `GET /api/admin/users` - List users (pagination)
- `POST /api/admin/users` - Create user
- `PUT /api/admin/users/:id` - Update user
- `DELETE /api/admin/users/:id` - Delete user
- `GET /api/admin/stats` - Dashboard statistics
- `GET /api/admin/sms` - List SMS audit logs (filterable, paginated)
- `POST /api/admin/sms` - Send manual SMS
- `PUT /api/admin/settings` - Update notification settings (encrypted)

**Locations**
- `GET /api/locations` - List locations
- Full CRUD if implemented

**Webhooks**
- `POST /api/webhook/twilio/inbound` - Inbound SMS webhook
- `POST /api/webhook/twilio/status` - Twilio status callback

For each endpoint include:
- Method & path
- Required role
- Request body schema (with types)
- Response schema (success & error examples)
- curl examples
- Notes (idempotency, rate limits, etc.)

Also include:
- Error code reference (400, 401, 403, 404, 409, 500)
- JWT usage (Authorization: Bearer <token>)
- Rate limiting info (100 requests per 15min per IP)
